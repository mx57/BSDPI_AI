## 2026-06-18 - [Optimization of AI UI and Registry]
**Инсайт:** Частые обновления UI (например, списка стратегий ИИ) через полный сброс коллекции (`Clear()` + `Add()`) вызывают мерцание и лишнюю нагрузку на GC. Блокировки в реестре на каждую строку списка приводили к O(N) операциям с локами.
**Действие:** Использовать дифференциальное обновление коллекции в ViewModel и групповые снимки данных (snapshots) из реестра под одним локом. Это снижает накладные расходы на синхронизацию и делает UI плавным.
## 2026-06-18 - [Optimization & Reliability]
**Инсайт:** Синхронные вызовы `Process.Kill()` и `WaitForExit()` в UI-потоке приводили к заметным «зависаниям» интерфейса при остановке нескольких процессов (Zapret + ByeDPI/Warp). Менеджер доменов игнорировал список исключений при синхронизации с движком.
**Действие:** Перевести все команды управления процессами в `async Task`. Использовать `WaitForExitAsync()` для неблокирующего ожидания завершения. Реализовать полную синхронизацию списков (цели + исключения) в `SyncCustomHostlist`. Внедрить debounced-сохранение настроек для предотвращения избыточной дисковой активности.
